- 新增正式工具 `autodokit/tools/aok_pdf_aliyun_multimodal_parse.py`，作为阿里百炼多模态 PDF 单篇解析入口。

## 2026-03-19

### 事务库合规化迁移与运行时改造

- **变更概述**：将官方事务改造成纯业务三件套（py / json / md），把引擎相关的运行时元数据从事务目录中剥离，集中维护在引擎注册表与覆盖文件中（非破坏性 overrides）。
- **关键实现**：
	- 修正并统一引擎默认事务数据库路径为 `autodo-engine/config` 下的注册表文件；
	- 扩展事务注册表记录字段（包含 `record_uid`、`display_name`、`folder_name`、`source_py_path`、`params_json_path`、`doc_md_path`、`node_template` 等）；
	- 将 flow 节点模板加载改为优先从运行时注册表（database-first），并保留基于事务包 manifest 的回退逻辑；
	- 新增 `import_user_affair` 管理器与命名冲突规则（自动 _vN 重命名策略）；
	- 增加批量迁移脚本 `migrate_affairs_to_compliance.py`，将 legacy manifest 拆分为纯业务参数与运行时 overrides。
- **迁移与验证结果**：
	- 对官方事务库执行迁移（dry-run + 实际执行），触及事务数量：85 个；
	- 生成并写入 `autodo-engine/config/affair_metadata_overrides.json`，以保存引擎关注的元数据；
	- 重建事务手册：`docs/AOK预置事务手册.md`（由覆盖元数据 + `affair.md` + 纯业务参数合成）；
	- 相关单元/集成测试通过（示例运行：9 个测试通过）；
	- 刷新运行时注册表后：记录数 85，错误 0，警告 0。
- **风险与注意事项**：
	- 元数据改为 overrides 后为非破坏性操作，运行时优先读取 DB 覆盖；手工/脚本写覆盖文件时请谨慎并保留历史备份；
	- 迁移后务必在 CI/部署前运行一致性校验（见后述计划）。
- **后续计划（建议优先级）**：
	- 实现一致性校验命令：三件套完整性、数据库悬挂引用检查、版本链连续性；
	- 提供一键重建脚本（rebuild registry + regenerate manual）；
	- 完善导入的事务化/回滚策略与 API 文档。
