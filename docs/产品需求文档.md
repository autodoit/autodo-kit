# 产品需求文档

## 1. 定位

autodo-kit 是 AOK 的官方事务内容库，目标是沉淀稳定可复用的事务实现、事务工具与事务文档，而不是承载调度引擎。

## 2. 本阶段目标

1. 维护官方 affairs 与 tools 的清晰目录边界。
2. 让官方事务可被 autodo-engine 稳定发现、注册与执行。
3. 通过事务手册与事务元数据保持内容可检索、可维护。

## 3. In Scope

- 官方事务实现
- 事务侧工具函数
- 事务元数据与事务说明文档
- 事务手册生成脚本

## 4. Out of Scope

- 流程图编译与调度
- 任务数据库与审计视图
- CLI 主入口
- 决策部门与状态机

## 5. 关键约束

1. 事务业务代码尽量不耦合引擎内部目录。
2. 路径处理统一走运行前预处理。
3. runner.module 与文档路径保持稳定，避免破坏现有事务注册链路。
