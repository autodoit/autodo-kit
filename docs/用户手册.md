# 用户手册

autodo-kit 用于提供官方事务与事务工具，不负责独立运行流程。典型使用方式是：由 autodo-engine 负责调度，本仓库负责提供可执行事务。

补充说明：

- AOB 迁移过来的执行型脚本入口现在由 `scripts/aob_tools/` 承接；
- 若你仍从 `autodo-lib/tools/*.py` 或 `autodo-lib/scripts/run_*.py` 调用，当前会自动转发到 AOK 新入口。

## 1. 安装

```powershell
uv pip install -e ../autodo-engine
uv pip install -e .
```

## 2. 查看与更新事务手册

```powershell
python scripts/generate_affair_manual.py
```

生成结果位于 docs/AOK预置事务手册.md。

## 3. 直调事务

推荐通过桥接入口调用：

```python
from pathlib import Path
import autodokit as aok

outputs = aok.run_affair(
    "PDF文件转md文件",
    config={
        "pdf_dir": "data/input",
        "output_dir": "data/output",
    },
    workspace_root=Path.cwd(),
)
print(outputs)
```

说明：

- run_affair 的执行器来自 autodo-engine
- 事务源码与事务文档来自 autodo-kit，事务元数据由事务管理系统数据库统一管理
- 路径字段会在进入运行时前统一绝对化

## 4. 用户覆盖事务

用户自定义事务仍放在工作区下列位置：

- .autodokit/affairs/
- .autodokit/affair_registry.json

当自定义 business 事务与官方同名时，运行时会按覆盖规则合并；graph 域事务不允许覆盖官方图节点事务。

## 5. 常见问题

1. 直接 import autodokit 后找不到 run_affair：先确认 autodo-engine 已安装。
2. 事务执行时报路径错误：优先检查 workspace_root 与配置中的路径字段。
3. 手册内容过旧：重新执行 scripts/generate_affair_manual.py。

## 6. AOB 迁移脚本入口

当前 AOK 额外承接了 AOB 的脚本型能力，主入口如下：

```powershell
python scripts/aob_tools/aoc.py validate --input libs
python scripts/aob_tools/deploy.py --help
python scripts/aob_tools/library.py items sync --dry-run
```

这些入口默认优先操作同级 `autodo-lib`；也可通过环境变量 `AOB_REPO_ROOT` 或命令参数 `--repo-root` 显式指定 AOB 仓库根目录。

## 7. 公开工具直调

用户侧推荐直接调用工具函数，不需要统一抽象入口：

```python
from autodokit.tools import parse_reference_text

result = parse_reference_text("Zhou, 2024. A research paper title...")
print(result)
```

你可以先查看公开工具清单，再按函数名调用：

```python
from autodokit.tools import list_user_tools

print(list_user_tools())
```

说明：

- 公开工具可直接获得 IDE 形参提示与自动补全。
- 公开工具统一由 `autodokit.tools.__init__` 管理并导出。
- 需要更底层辅助能力时，请参考开发者指南中的“开发者工具”。

## 8. 事务目录合规要求

- 每个事务目录只保留 `affair.py`、`affair.json`、`affair.md` 三个核心文件。
- `affair.json` 只存放纯业务参数模板，不再存放 runner/domain/node 等引擎元数据。
- 若新增事务，推荐使用 `autodokit.import_user_affair(...)` 自动导入并注册。

## 9. 工具直调示例（demos）
